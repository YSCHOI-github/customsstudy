## 1. UCP 600 제17조 — 원본서류 및 사본 ★2025년 기출

### ① 원칙 (제17조(a))
- 신용장에 명시된 서류는 각 서류마다 **원본 1통 이상** 제시되어야 함

### ② 원본서류 인정기준 (제17조(b))
- 은행은 서류 발행인이 원본이라고 표시한 서류를 원본으로 취급함

### ③ 원본 표시 없는 경우 원본 인정요건 (제17조(c)) ★2025년 직접 출제
원본으로 인정되는 경우 — 아래 중 하나를 충족하면 원본으로 취급됨:

1. 서류가 발행인의 **자필서명·수기·퍼포레이션·스탬프·라벨·직인** 등으로 표시된 경우
2. 서류가 발행인의 **원본 레터헤드(Original Letterhead)** 상에 작성된 것으로 보이는 경우
3. 서류가 **사본(copy)이라고 표시되지 않은** 경우

### ④ 사본(Copy) 처리
- "사본 1통" 요구 시 → 원본 또는 사본 모두 수리 가능함

## 2. UCP 600 제18조 — 상업송장 (Commercial Invoice) ★2025년 기출

### ① 발행요건 (제18조(a)) ★2025년 직접 출제

1. **발행인**: 수익자(Beneficiary)가 발행하여야 함
2. **수신인**: 개설의뢰인(Applicant) 앞으로 발행되어야 함
   - 단, 개설의뢰인 이외 당사자 앞 발행 불가 (양도가능신용장의 경우 예외)
3. **통화**: 신용장과 **동일 통화**로 표시되어야 함
4. **금액**: 신용장 금액을 **초과하지 않아야** 함
5. **물품·용역·이행 명세**: 신용장에 기재된 명세와 **일치**하여야 함

### ② 비교 — 운송서류·보험서류와의 서명요건 차이

| 서류 | 서명 필수 여부 |
|---|---|
| 상업송장 (제18조) | **서명 불필요** |
| 선하증권 (제20조) | 서명 필수 |
| 복합운송서류 (제19조) | 서명 필수 |
| 보험서류 (제28조) | 서명 필수 |

## 3. UCP 600 제19조 — 복합운송서류 ★★★2020년 30점 + 2025년 기출

### ① 서명요건 (제19조(a)(i)) ★2025년 직접 출제

서류는 다음 중 하나에 의해 **서명**되어야 함:
1. **운송인(Carrier)** 또는 운송인을 대리하는 **지정 대리인(Named Agent)**
2. **선장(Master)** 또는 선장을 대리하는 **지정 대리인**

→ 대리인 서명 시: 대리인이 **운송인을 대리하는지 또는 선장을 대리하는지** 명시해야 함

### ② 복합운송서류 수리요건 6가지 (제19조(a)) ★2020년 30점 전항 출제

1. **서명요건**: 운송인·선장 또는 각각의 지정 대리인 서명
2. **화물 수취 표시**: 수취일 또는 발송일 명시 → 이 일자가 선적일로 간주됨
3. **발송지**: 신용장에 명시된 물품 수취·발송·선적지가 기재되어야 함
4. **최종목적지**: 신용장에 명시된 최종 목적지가 기재되어야 함
5. **유일한 원본 또는 전통(Full Set)**: 원본 전통이 발행된 경우 전통 제시
6. **운송조건 포함 또는 참조**: 약식(Short Form)·이면백지(Blank Back) 복합운송서류도 수리 가능

### ③ 환적 (제19조(b)) ★2025년 직접 출제

- **환적(Transshipment)**: 발송지~최종목적지 운송 도중 **한 운송수단에서 다른 운송수단으로 화물 이환**하는 것임
- 신용장이 환적을 금지하더라도, 운송 전 구간이 **단일 복합운송서류**로 커버되는 경우 환적 표시 서류도 수리 가능함

## 4. UCP 600 제20조 — 선하증권 (Bill of Lading; B/L) ★★★핵심

### ① 선하증권의 3대 기능

1. **화물수취증(Receipt for Goods)**: 선사가 화물 수취를 증명하는 서류
2. **운송계약의 증거(Evidence of Contract of Carriage)**: 화주-선사 간 계약 내용 증명
3. **유통성 권리증권(Negotiable Document of Title)**: 배서 양도로 화물 처분권 이전 가능 → 무역금융 담보 역할

### ② UCP 600 제20조 수리요건 (a항 전체)

| 번호 | 요건 | 세부 내용 |
|---|---|---|
| ① | **서명요건** | 운송인 또는 지정 대리인, 선장 또는 지정 대리인 서명 (대리인 시 대리 상대방 명시 필수) |
| ② | **선적 표시** | "본선 적재(On Board)" 표시 + 선적일 명시 (수취식 B/L은 본선적재부기 필요) |
| ③ | **선적항** | 신용장에 명시된 선적항이 기재되어야 함 |
| ④ | **양륙항** | 신용장에 명시된 양륙항이 기재되어야 함 |
| ⑤ | **원본** | 발행된 원본 전통(Full Set) 제시 |
| ⑥ | **용선계약 비기초** | 용선계약에 기초한 B/L이 아닐 것 (→ 제22조 별도 규정) |

### ③ 환적 (제20조(b)~(c))

- **원칙**: 신용장이 환적 금지 명시 시 환적 표시 B/L 수리 불가
- **예외**: 환적이 **신용장에 명시된 선적항~양륙항 전 구간을 단일 B/L로 커버**하는 경우 환적 금지 조항에도 불구하고 수리 가능함

### ④ 선하증권의 종류 비교

| 분류기준 | 종류 | 내용 |
|---|---|---|
| **유통성** | 지시식 B/L (Order B/L) | 배서 양도 가능 — 무역금융·L/C 거래 표준 |
| | 기명식 B/L (Straight B/L) | 특정인 지정 — 양도 불가, 비유통성 |
| **적재 여부** | 선적식 B/L (Shipped/On Board B/L) | 실제 선적 완료 후 발행 — L/C 거래 요구 |
| | 수취식 B/L (Received B/L) | 화물 수취만으로 발행 → 본선적재부기 필요 |
| **사고 여부** | 무사고 B/L (Clean B/L) | 화물 상태 이상 없음 표시 — L/C 수리 조건 |
| | 사고부 B/L (Foul/Dirty B/L) | 화물 이상 부기 → 원칙적 수리 불가 |
| **발행자** | 선사 B/L (Master B/L) | 실제 운송인(선사) 발행 |
| | 포워더 B/L (House B/L) | 운송주선인 발행 — L/C에서 허용 여부 명시 필요 |
| **거래유형** | 제3자 B/L (Third Party B/L) | 수익자(중간상)가 아닌 제3자(실제 공급자)가 발행 |
| | 스위치 B/L (Switch B/L) | 원 B/L 회수 후 새 B/L 발행 — 중계무역 활용 |

### ⑤ 제3자 B/L vs 스위치 B/L ★2019년 제36회 10점 기출

**제3자 선화증권 (Third Party B/L)**
- 정의: 신용장의 수익자(중간 수출상)가 아닌 **제3자(최초 공급자·제조업자)** 명의로 발행된 B/L
- 활용: 중계무역에서 최초 공급자의 정보 노출 허용 시 사용됨
- UCP: 신용장에 명시적으로 허용하지 않아도 수리 가능함 (발행인 명칭 불문)

**스위치 선화증권 (Switch B/L)**
- 정의: 이미 발행된 원 B/L 전통을 회수하고, **새로운 내용(선적인·가격·목적지 등 변경)으로 교체 발행**한 B/L
- 활용: 중계무역에서 최초 공급자·원산지·가격 등 정보 은폐 목적으로 사용됨
- 절차: 원 B/L 전통 회수 → 스위치 B/L 발행 → 원 B/L 효력 소멸

| 구분 | 제3자 B/L | 스위치 B/L |
|---|---|---|
| 발행 시점 | 최초 선적 시 | 원 B/L 발행 후 교체 |
| 원 B/L 회수 여부 | 불필요 | **원 B/L 전통 반드시 회수** |
| 정보 은폐 정도 | 부분적 (수익자 노출 허용) | 완전 은폐 가능 |
| 사용 목적 | 공급자 명의 유지 | 중계무역 정보 은폐 |

### ⑥ 선하증권 조항 영문 작성 ★2023년 제40회 10점 기출

**작성조건**: FOB Busan Port, Korea, Incoterms® 2020, ABC은행 지시식

```
Full set of clean on board ocean Bills of Lading made out to order of
ABC Bank, notify Applicant, marked "Freight Collect",
Port of Loading: Busan Port, Korea
```

**조항 해설**

| 표현 | 의미 |
|---|---|
| Full set | 발행된 원본 전통 전부 (통상 3통) |
| Clean on board | 무사고·선적식 B/L |
| Made out to order of ABC Bank | ABC은행 지시식 (지시식 B/L) |
| Notify Applicant | 도착통지처: 개설의뢰인(수입자) |
| Freight Collect | 운임 미지급 (수입자 부담) ← FOB이므로 |
| Port of Loading: Busan Port, Korea | FOB 조건상 선적항 명시 |

## 5. UCP 600 제21조 — 비유통성 해상화물운송장 (Non-Negotiable Sea Waybill)

### ① B/L과의 핵심 차이

| 구분 | 선하증권 (B/L) | 해상화물운송장 (SWB) |
|---|---|---|
| 유통성 | **유통성 권리증권** (배서 양도) | **비유통성** — 양도 불가 |
| 화물 인도 | 원본 제시 必 | 원본 없이 수하인 확인만으로 인도 |
| 결제방식 | L/C 거래 표준 | 본·지사 간 거래, 신뢰관계 거래 |
| 사기 위험 | 낮음 | 상대적으로 높음 |

### ② 수리요건 (제21조(a))
- 서명요건: 제20조(a)(i)와 동일 (운송인·선장 또는 지정 대리인)
- "본선 적재(On Board)" 표시 + 선적일 명시
- 선적항·양륙항 기재

## 6. UCP 600 제22조 — 용선계약선하증권 (Charter Party B/L)

### ① 개념
- 용선계약(Charter Party)에 기초하여 발행된 B/L
- **제20조(a)(vi)에 의해 일반 신용장에서는 수리 불가** → 신용장에 명시적 허용 시에만 수리

### ② 수리요건 (제22조)
1. 서명요건: 선장·선주(Owner)·용선자(Charterer) 또는 각각의 지정 대리인 서명
2. 용선계약 사본 제시 불필요 (단, 신용장이 요구하는 경우 제시)
3. 선적일·선적항·양륙항 기재

## 7. UCP 600 제23조 — 항공운송서류 (Air Transport Document; AWB) ★2018년 15점 기출

### ① 항공화물운송장(AWB)의 기능

1. **운송계약의 증거**: 항공사-화주 간 계약 내용 증명
2. **화물수취증**: 화물 수취 확인
3. **운임청구서**: 운임·부대비용 기재
4. **세관신고서**: 통관 서류로 활용
5. **보험증명서**: 부보 시 증명 역할
6. **운송지시서**: 운송 취급 지시 포함
- ⚠️ **비유통성(Non-Negotiable)** — 유가증권 기능 없음 (B/L과 핵심 차이)

### ② UCP 600 제23조 수리요건 ★2018년 15점 기출

1. **서명요건**: 항공운송인(Carrier) 또는 지정 대리인 서명 — 대리인 서명 시 운송인 대리 명시 필수
2. **실제 출발일(Actual Date of Dispatch)**: 발행일이 아닌 **실제 출발일이 선적일**로 간주됨
   - "예약일(Date of Issue)"이 기재된 경우: 실제 출발일을 별도 명시하지 않으면 발행일 = 선적일
3. **출발지 공항**: 신용장 명시 출발지 공항 기재
4. **도착지 공항**: 신용장 명시 도착지 공항 기재
5. **원본 요건**: 수하인(Consignee) 원본 = 3통 중 제3원본(For Shipper) 제시
6. **운송조건**: 약식·이면백지 AWB도 수리 가능

## 8. UCP 600 제24조 — 도로·철도·내륙수로운송서류 ★2023년 10점 기출

### ① 서명요건 (제24조(a)(i)) ★2023년 직접 출제
- 도로·철도·내륙수로 **운송인(Carrier)** 또는 지정 대리인이 서명·스탬프·인증하여야 함
- 대리인 서명 시: 운송인을 대리한다는 명시 필요

### ② 선적일 판단기준 (제24조(a)(ii)) ★2023년 직접 출제
- **철도**: 운송서류의 **스탬프 날인일** 또는 **서명일** = 선적일
- **도로·내륙수로**: 화물 **수취일(Date of Receipt)** = 선적일
- 별도 수취일 미기재 시: 운송서류 **발행일** = 선적일로 간주

### ③ 원본요건 (제24조(b)) ★2023년 직접 출제
- 운송서류가 **원본 1통만 발행**된 경우 → 원본 1통 제시로 충족
- **복수 원본** 발행된 경우 → **전통(Full Set) 제시** 요구됨
- 철도화물운송장(Railway Consignment Note): 화주 보관 원본 1통 제시로 충족

## 9. UCP 600 제25조 — 특송수령증·우편수령증·우송증명서 ★2025년 기출

### ① 특송수령증 (Courier Receipt) 서명요건 (제25조(a)) ★2025년 직접 출제

1. **서명요건**: 특송업자(Courier) 명칭이 기재되어야 하고, 특송업자 또는 지정 대리인이 **서명·스탬프·인증**하여야 함
2. **수취일(Date of Receipt)**: 수취일 = 선적일로 간주
3. **수취장소**: 신용장에 명시된 장소에서 수취됨을 명시

### ② 우편수령증·우송증명서 (Post Receipt / Certificate of Posting) (제25조(b)~(c))
- 우편관청(Post Office) 또는 우편서비스업자가 **스탬프·서명**하여야 함
- 스탬프 날인일 = 선적일

## 10. UCP 600 제28조 — 보험서류 및 보험증권 ★2019년 15점 기출

### ① 수리요건 (제28조 전체) ★2019년 15점 전항 출제

| 번호 | 요건 | 세부 내용 |
|---|---|---|
| ① | **발행인** | 보험회사(Insurance Company), 보험업자(Underwriter) 또는 각각의 대리인·수권인 서명 |
| ② | **원본** | 원본 보험서류 전통(Full Set) 제시 (2통 이상 발행 시 전통) |
| ③ | **부보금액** | **CIF 또는 CIP 가격의 최소 110%** 이상 부보 (표시통화는 신용장과 동일) |
| ④ | **부보시점** | 선적일 이전 또는 선적일 → 단, 서류 발행일이 선적일 이후인 경우 면책기간 없이 선적일부터 소급 적용됨을 명시 |
| ⑤ | **부보조건** | 신용장 명시 조건 (불명시 시: 서류상 일치 조건으로 수리) |
| ⑥ | **서류 유형** | 보험증권(Policy) 또는 확정통지서(Declaration) 수리 가능 — 보험계약증서(Cover Note)는 수리 불가 |

### ② 보험서류 유형 비교

| 서류명 | 수리 가능 여부 | 특징 |
|---|---|---|
| 보험증권 (Insurance Policy) | ✅ 수리 가능 | 개별 계약 건별 발행 |
| 확정통지서 (Declaration) | ✅ 수리 가능 | 포괄예정보험 하에서 발행 |
| 보험계약증서 (Cover Note) | ❌ **수리 불가** | 보험중개인 발행 임시 문서 — 정식 보험 아님 |
| 예정보험증권 (Open Policy) | ✅ 수리 가능 | 포괄예정보험 증권 |

## 11. 상업송장 외 주요 무역서류

### 11-1. 원산지증명서 (Certificate of Origin; C/O)

| 구분 | 일반 원산지증명서 | FTA 원산지증명서 |
|---|---|---|
| 발급 기관 | 상공회의소·세관 | 기관발급(세관·상공회의소) 또는 자율발급(인증수출자) |
| 목적 | 원산지 확인·통관 | FTA 특혜관세 적용 |
| 양식 | 일반 양식 | FTA별 지정 양식 (GSP Form A 등) |
| 유효기간 | 없음 | 협정별 상이 (통상 1~2년) |

### 11-2. 포장명세서 (Packing List)
- 개념: 각 포장 단위별 품명·수량·중량·용적 상세 기재
- 기능: 세관 심사·수하인 확인·클레임 증빙용
- UCP: 서류에 요구 시 발행인 특정 없으면 수익자 이외 당사자도 수리 가능

### 11-3. 검사증명서 (Inspection Certificate)
- 개념: 제3자 검사기관이 물품 품질·수량·포장 상태 확인 후 발행
- 주요 발급기관: SGS, Bureau Veritas, Intertek 등

### 11-4. 영사송장 (Consular Invoice)
- 개념: 수입국 주재 영사관이 인증·발급하는 특수 송장
- 목적: 수입국의 가격 덤핑·원산지 위장 방지
- 사용국: 남미·중동 일부 국가

## 12. EDI / 전자무역

### 12-1. EDI (Electronic Data Interchange) 개념
- 기업 간 구조화된 전자문서를 **표준화된 포맷**으로 컴퓨터 간 직접 교환하는 시스템
- 무역 EDI: 선적서류·세관신고·운임청구 등을 전자적으로 처리

### 12-2. eUCP (UCP 600 Supplement for Electronic Presentation)
- 개념: UCP 600의 전자적 제시를 위한 보충 규칙 (2019년 eUCP v2.0 시행)
- 적용: 신용장에 eUCP 적용 명시 시 UCP 600과 함께 적용됨
- "전자기록(Electronic Record)": 종이서류 대신 전자적 방법으로 작성·전송·제시되는 자료

### 12-3. 전자선하증권 (e-B/L)
- 개념: 종이 B/L을 전자적으로 대체한 디지털 권리증권
- 플랫폼: BOLERO, essDOCS, TradeLens 등
- 법적 근거: 각국 전자거래법·UNCITRAL 전자양도성기록모델법

### ▶ 선하증권 수리요건 6가지 — 서·선·항·항·원·용

1. **서**(명요건): 운송인·선장 또는 지정 대리인
2. **선**(적표시): On Board + 선적일
3. **항**(구): 선적항 기재
4. **항**(구): 양륙항 기재
5. **원**(본): 전통 제시
6. **용**(선계약 비기초): 용선계약 B/L 아닐 것

### ▶ 보험서류 부보금액 — "110% 이상"
- CIF·CIP 가격의 **최소 110%** — 단 1점 감점 주의
